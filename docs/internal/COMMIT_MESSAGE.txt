Replaced print-based logging with a structured logging framework to improve
debugging, operations, and code quality.

## What Changed

### Core Framework
- Added `scripts/logging_config.py` with:
  * Multiple log levels (DEBUG, INFO, WARNING, ERROR, CRITICAL)
  * Colored console output (with colorama, optional)
  * Structured logging format (key=value pairs)
  * File logging support with rotation
  * Context managers for adding structured fields
  * Easy CLI integration helpers

### Migrated Files (100%)
- `scripts/operations.py` (15 print statements → logger calls)
  * All 8 operation classes
  * Proper log levels for skips (DEBUG), operations (INFO), errors (ERROR)

  * Structured context fields (path, mode, ownership, etc.)

## Benefits

### For Developers
- Exception tracebacks automatically included in logs
- Structured context fields for debugging
- Easy to filter by severity
- Testable (can mock logger)

### For Users
- Colored output improves readability
- Can control verbosity with LOGLEVEL env var
- Clearer error messages with context
- Automation-friendly structured format (STRUCTURED=true)

### For Operations
- File logging for troubleshooting
- Machine-parseable structured logs
- Integration-ready for monitoring systems

## Usage Examples

### For scripts:
```python
from logging_config import get_logger, setup_logging

logger = get_logger(__name__)

def main():
    setup_logging(level="INFO", enable_colors=True)
    logger.info("Operation started", extra={"service": "adguard"})
```

### For users:
```bash
# Verbose output
LOGLEVEL=DEBUG make enable-service adguard

# Structured output for automation
STRUCTURED=true make backup > backup.log
```
